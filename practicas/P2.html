2 ¿Qué es un dataframe y cómo se diferencia de una “base de datos”?

2.1 El dataframe en R

Un dataframe es la estructura de R para almacenar datos : las filas son observaciones o casos y las columnas son variables. A diferencia de las matrices, cada columna puede tener un tipo diferente (numérico, carácter, lógico, etc.), pero todas las columnas deben tener el mismo número de filas (aunque algunas puedan tener casos perdidos).

Podemos construir un pequeño data frame “a mano”.

df_ejemplo <- data.frame(
  id       = 1:4,
  entidad  = c("Morelia", "Marcos Castellanos", "Lázaro Cardenas", "Uruapan"),
  imn_2020 = c(0.93, 0.92, 0.91, 0.91),
  pob  = c(849053, 13983, 196003, 356786
)
)

df_ejemplo
id entidad imn_2020 pob
1 Morelia 0.93 849053
2 Marcos Castellanos 0.92 13983
3 Lázaro Cardenas 0.91 196003
4 Uruapan 0.91 356786

En {tidyverse}, la versión del dataframe se llama tibble (tbl_df). Es prácticamente igual pero tiene impresión más limpia y mensajes de advertencia más útiles:

library(tidyverse)

tb_ejemplo <- tibble(
  id       = 1:4,
  entidad  = c("Morelia", "Marcos Castellanos", "Lázaro Cardenas", "Uruapan"),
  imn_2020 = c(0.93, 0.92, 0.91, 0.91),
  pob  = c(849053, 13983, 196003, 356786))

Veamos cómo se ve

tb_ejemplo
id entidad imn_2020 pob
1 Morelia 0.93 849053
2 Marcos Castellanos 0.92 13983
3 Lázaro Cardenas 0.91 196003
4 Uruapan 0.91 356786

¿Y si vemos el tipo?

class(tb_ejemplo)
[1] "tbl_df"     "tbl"        "data.frame"

4 Importación de distintos formatos

4.1 CSV con read_csv()

El formato CSV refiere a archivos con valores separados por comas.

Podemos leer el CSV desde la ruta dentro del proyecto.

concentrado <- read_csv(here("data", "raw", "concentradohogar.csv"))
NotaCSV con punto y coma

En algunos sistemas (especialmente Windows en español), Excel guarda los CSV con punto y coma como separador. En ese caso se usa readr::read_csv2() en lugar de read_csv().

4.2 Excel con readxl::read_excel()

Vamos a importar datos del IMCO.

imco <- read_excel(here("data", "raw", "competitividad_estatal_2025.xlsx"))

Podemos ver las hojas disponibles de nuestro archivo.

excel_sheets(here("data", "raw", "competitividad_estatal_2025.xlsx"))

4.3 Stata con haven::read_dta()

Importemos una base de datos del Barómetro de las Américas.

lapop <- read_dta(here("data", "raw", "mex_lapop_2023.dta"))
TipFuentes nacionales

El INEGI publica microdatos en formato Stata, csv, excel, dbf, R para la ENOE, ENIGH, ENVIPE, ENDIREH, entre otras. Los archivos suelen encontrarse dentro de carpetas comprimidas .zip descargadas del portal del INEGI.

4.4 Parquet con arrow::read_parquet() (opcional)

Parquet es un formato columnar de alta eficiencia, útil cuando los archivos CSV son muy grandes (millones de registros). Algunos repositorios de datos abiertos ya lo adoptan.

Nosotres aquí haremos una pruebita y primero vamos a convertir “concentradohogar” en formato parquet y luego leerlo.

write_parquet(concentrado, here("data", "raw", "concentradohogar.parquet"))
concentrado <- read_parquet(here("data", "raw", "concentradohogar.parquet"))

5 Limpieza de nombres de columnas con janitor::clean_names()

Cuando importamos datos de fuentes externas, los nombres de columnas suelen tener espacios, acentos, mayúsculas o caracteres especiales que complican su uso en R. clean_names() los normaliza.

Podemos aplicar clean_names() inmediatamente después de importar cualquier tabla, antes de manipularla.

lapop <- janitor::clean_names(lapop)

6 Inspección rápida de un dataframe

Antes de analizar cualquier dataframe, debemos hacer una revisión inicial para entender su estructura y detectar posibles problemas.

6.1 Dimensiones y nombres de columnas

dim(concentrado)          # Filas y columnas
nrow(concentrado)         # Solo filas
ncol(concentrado)         # Solo columnas
names(concentrado)        # Nombres de las columnas

6.2 Clases por columna

str(concentrado)          # Estructura completa: tipo + primeros valores
glimpse(concentrado)      # Versión tidyverse: más legible

6.3 Detección de NAs

colSums(is.na(lapop))
Advertencia¿Por qué importa detectar NA desde el inicio?

Los valores faltantes (NA) pueden alterar los resultados de funciones como mean(), sum() o modelos de regresión. Identificarlos al inicio del análisis evita errores.

7 Exportar de resultados

7.1 Exportar a CSV con write_csv()

write_csv(concentrado, here("outputs", "tables", "prueba1.csv"))

7.2 Exportar a Excel con writexl::write_xlsx()

{writexl} no requiere tener Excel instalado.

write_xlsx(concentrado, here("outputs", "tables", "prueba2.xlsx"))

8 Buenas prácticas: nombres de archivos

El nombre de un archivo de datos es importante ya que se puede facilitar la reproducibilidad y el trabajo colaborativo.

8.1 Reglas esenciales

  • Usar guiones bajos _ o guiones -
  • Fechas en formato AAAA_MM_DD
  • Todo en minúsculas

8.2 Estructura de carpetas recomendada (recordatorio)

mi_proyecto/
├── data/
│   ├── raw/              ← Archivos originales (nunca modificar)
│   └── processed/        ← Datos ya limpios y transformados
├── scripts/              ← Archivos .R y .qmd
├── outputs/
│   ├── figures/
│   └── tables/
└── README.md
