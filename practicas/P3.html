4 Tipos de variables y codificación

Antes de crear o recodificar variables, debemos ver qué tipo de variable tenemos, ya que esto influye en qué funciones debemos usar.

4.1 Por ejemplo

Tipo Descripción Ejemplo en ENIGH
Nominal Categorías sin orden Sexo, entidad federativa
Ordinal Categorías con orden Nivel educativo, estrato
Discreta Valores enteros contables Número de integrantes del hogar
Continua Valores numéricos en un rango Ingreso corriente, gasto

4.2 Tipos en R (repaso)

En R, estas categorías se traducen en las siguientes:

  • numeric / integer: variables cuantitativas (continuas o discretas).
  • character: cadenas de texto, normalmente variables nominales sin orden.
  • factor: variables categóricas con niveles definidos (nominales u ordinales).
  • logical: variables lógicas (TRUE/FALSE), equivalentes a dicotómicas.

Podemos revisar el tipo de cualquier variable con class():

class(concentrado$ing_cor)
class(concentrado$educa_jefe)

5 Creación de variables con mutate()

mutate() agrega o modifica columnas dentro de un dataframe. El input, digamos, es cualquier expresión válida de R: operaciones aritméticas, funciones estadísticas, funciones de texto, etc.

5.1 Cálculos numéricos

La variable ing_cor refiere al ingreso corriente trimestral del hogar. Podemos calcular por mes o el logaritmo.

concentrado <- concentrado %>%
  mutate(
    ing_mensual  = ing_cor / 3,            # ingreso promedio mensual
    log_ing      = log(ing_cor)
  )
TipVarias variables en un solo mutate()

Podemos crear todas las variables que necesitemos dentro de un solo bloque mutate(), separadas por comas.

6 Recodificación con if_else()

if_else() es la función más utilizada para crear variables dicotómicas: si se cumple una condición, asigna un valor; si no, asigna otro.

if_else(condición, valor_si_verdadero, valor_si_falso)

6.1 Por ejemplo: persona jefa del hogar mayor de 65 años

Primero creamos la variable.

concentrado <- concentrado %>%
  mutate(jefe_mayor65 = if_else(edad_jefe >= 65, "65 años o más", "Menos de 65 años"))

Luego especificamos los niveles

concentrado <- concentrado %>%
  mutate(jefe_mayor65 = factor(jefe_mayor65,
  levels = c("Menos de 65 años", "65 años o más")))

Luego la vemos

concentrado %>% tabyl(jefe_mayor65)

7 Recodificación con case_when()

case_when() evalúa cada condición y asigna el valor de una variable nueva a partir de una antigua. Es como una serie de “si se cumple esto, asigna esto”.

mutate(nueva_var = case_when(
  condición1 ~ "valor1",
  condición2 ~ "valor2",
  TRUE        ~ NA_character_   # para los casos que no cumplen ninguna condición
))

7.1 Ejemplo: rangos de edad de la persona jefa del hogar

concentrado <- concentrado %>%
  mutate(
    rango_edad_jefe = case_when(
      edad_jefe >= 18 & edad_jefe <= 29 ~ "18-29",
      edad_jefe >= 30 & edad_jefe <= 44 ~ "30-44",
      edad_jefe >= 45 & edad_jefe <= 59 ~ "45-59",
      edad_jefe >= 60 & edad_jefe <= 74 ~ "60-74",
      edad_jefe >= 75                   ~ "75 o más",
      TRUE                              ~ NA_character_
    )
  )

concentrado %>% tabyl(rango_edad_jefe)

Convertimos a factor y especificamos niveles:

concentrado <- concentrado %>%
  mutate(
    rango_edad_jefe = factor(rango_edad_jefe,
                             levels = c("18-29", "30-44", "45-59","60-74", "75 o más")))

7.2 Ejemplo: nivel de escolaridad del jefe o jefa del hogar

La variable educa_jefe sigue la siguiente clasificación del INEGI:

Código Descripción
1 Sin instrucción
2 Preescolar
3 Primaria incompleta
4 Primaria completa
5 Secundaria incompleta
6 Secundaria completa
7 Preparatoria incompleta
8 Preparatoria completa
9 Profesional incompleta
10 Profesional completa
11 Posgrado

Agrupamos estos códigos en tres grandes niveles:

concentrado <- concentrado %>%
  mutate(
    nivel_educa = case_when(
      educa_jefe %in% c("01", "02", "03", "04") ~ "Básica o menos",
      educa_jefe %in% c("05", "06", "07", "08") ~ "Media superior",
      educa_jefe %in% c("09", "10", "11")        ~ "Superior",
      TRUE                                        ~ NA_character_
    ),
    nivel_educa = factor(nivel_educa, levels = c("Básica o menos", "Media superior", "Superior"))
  )

concentrado %>% tabyl(nivel_educa)

Verificamos que la recodificación sea correcta con una tabla cruzada:

table(concentrado$educa_jefe, concentrado$nivel_educa)

8 Recodificación con recode()

recode() es útil para hacer sustituciones directas de valores, sin necesidad de escribir condiciones. Funciona bien para variables con pocos valores distintos y conocidos.

8.1 Ejemplo: etiquetas para el estrato socioeconómico

La variable est_socio tiene cuatro categorías (1 a 4). Le asignamos etiquetas descriptivas:

concentrado <- concentrado %>%
  mutate(
    estrato_label = recode(est_socio,
      `1` = "Bajo",
      `2` = "Medio bajo",
      `3` = "Medio alto",
      `4` = "Alto",
      .default = NA_character_
    )
  )

concentrado %>% tabyl(estrato_label)
Tiprecode() vs case_when()

recode() es más conciso cuando se trata de sustituciones uno a uno con valores conocidos. case_when() es más flexible cuando las condiciones implican rangos numéricos, operadores lógicos (&, |) o combinaciones de varias variables.

9 Colapsar categorías con forcats::fct_lump_prop()

Cuando una variable categórica tiene muchas categorías poco frecuentes, conviene agruparlas en una categoría “Otros”.

9.1 Ejemplo: colapsar niveles poco frecuentes de educación

Primero convertimos a factor

concentrado$educa_jefe <- as.factor(concentrado$educa_jefe)

Revisamos las proporciones actuales

concentrado %>% tabyl(educa_jefe)

Colapsamos las categorías con menos del 10% de los casos

concentrado <- concentrado %>%
  mutate(
    educa_agrupada = fct_lump_prop(educa_jefe, prop = 0.10, other_level = "Otros")
  )

Revisamos cuáles quedaron.

concentrado %>% tabyl(educa_agrupada)

Una vez agrupadas las categorías con menos del 10%, podemos etiquetar las más frecuentes con fct_recode():

concentrado <- concentrado %>%
  mutate(
    educa_agrupada = fct_recode(educa_agrupada,
      "Primaria incompleta"  = "03",
      "Primaria completa"    = "04",
      "Secundaria completa"  = "06",
      "Preparatoria completa" = "08",
      "Profesional completa" = "10"
    )
  )

La vemos.

concentrado %>% tabyl(educa_agrupada)

10 Fechas

10.1 Extraer componentes de una fecha

La variable fecha_inicio viene como character. ¿Qué pasa si sólo tuviéramos la variable fecha_inicio y queremos extraer el año, mes y día para generar tres variables distintas? Antes de extraer cualquier componente, debemos convertirla al tipo Date con ymd(), que asume el orden Año-Mes-Día:

victimas <- victimas %>% 
  mutate(fecha_inicio = ymd(fecha_inicio))

La función ymd() de lubridate convierte en texto la fecha, pero asumiendo que el formato está en orden Year-Month-Day (como viene originalmente).

Ahora sí, queremos extraer el año, mes y día a partir de fecha_inicio.

victimas <- victimas %>% 
  mutate(
    annio = year(fecha_inicio),   # Año
    mes   = month(fecha_inicio),  # Mes (número)
    dia   = day(fecha_inicio)     # Día
  )

Los objetos Date en R se almacenan como números enteros (días transcurridos desde cierta fecha). El formato que vemos al imprimir es solo una representación. Esto significa que el orden día/mes/año no existe dentro del objeto; solo existe al momento de imprimirlo o exportarlo.

format() convierte una fecha a character con el orden deseado:

victimas <- victimas %>%
  mutate(
    fecha_formateada1 = format(fecha_inicio, "%d-%m-%Y"),  # día-mes-año
    fecha_formateada2 = format(fecha_inicio, "%m/%d/%Y"),  # mes/día/año
    fecha_formateada3 = format(fecha_inicio, "%Y/%m/%d")   # año/mes/día
  )

10.2 Cálculo de duraciones

No necesitamos cambiar formatos para hacer cálculos. difftime() calcula la diferencia entre dos fechas directamente sobre objetos Date:

# Convertir fecha_hecho a Date
victimas <- victimas %>%
  mutate(fecha_hecho = ymd(fecha_hecho))

# Calcular días entre el hecho y el inicio de la carpeta
victimas <- victimas %>%
  mutate(
    diferencia_dias = as.numeric(difftime(fecha_inicio, fecha_hecho, units = "days"))
  )

10.3 Creación de variables de fecha

Si tenemos el mes y el año en columnas distintas, podemos combinarlas con make_date(). Como esta función requiere al menos día, mes y año, asigna el día 1 por defecto cuando solo se proveen mes y año:

victimas <- victimas %>%
  mutate(fecha_mes_año = make_date(month = mes, year = annio))
# Reformatear para mostrar solo mes y año
victimas <- victimas %>%
  mutate(fecha_mes_año = format(fecha_mes_año, "%m/%Y"))

Si tenemos la fecha en una columna y la hora en otra, las unimos con paste() antes de convertir:

victimas <- victimas %>%
  mutate(fecha_hora = ymd_hms(paste(fecha_inicio, hora_inicio)))

class(victimas$fecha_hora)

11 (Optativo) Etiquetas provenientes de Stata con {haven}

Cuando los datos provienen de archivos .dta (Stata), las variables suelen traer etiquetas de valor (value labels). {haven} las preserva como atributos del objeto en R.

11.1 Leer el archivo y revisar etiquetas

Si tuviéramos que importar bases en formato Stata:

lapop_dta <- read_dta(here("data", "raw", "mex_lapop_2023.dta"))

Podemos ver las etiquetas de la variable

attr(lapop_dta$ur, "labels")

11.2 Convertir etiquetas a factor

as_factor() convierte las etiquetas Stata en niveles de factor en R

lapop_dta <- lapop_dta %>%
  mutate(ur = as_factor(ur))

Veamos los niveles.

levels(lapop_dta$ur)
Notaas_factor() de {haven}

A diferencia de as.factor() de R base, haven::as_factor() usa las etiquetas del archivo Stata como niveles del factor (en lugar de los valores numéricos). Esto facilita enormemente el trabajo con microdatos del INEGI distribuidos en formato Stata.

## Recodificación con `recode()`

`recode()` es útil para hacer **sustituciones directas** de valores, sin necesidad de escribir condiciones. Funciona bien para variables con pocos valores distintos y conocidos.

#| eval: false
concentrado <- concentrado %>%

## Colapsar categorías con `forcats::fct_lump_prop()`

Cuando una variable categórica tiene muchas categorías poco frecuentes, conviene agruparlas en una categoría "Otros".

### Ejemplo: colapsar niveles poco frecuentes de educación

Primero convertimos a factor
```{r}
#| eval: false
concentrado$educa_jefe <- as.factor(concentrado$educa_jefe)
```

Revisamos las proporciones actuales

```{r}
#| eval: false
concentrado %>% tabyl(educa_jefe)
```

Colapsamos las categorías con menos del 10% de los casos
```{r}
#| eval: false
concentrado <- concentrado %>%
  mutate(
    educa_agrupada = fct_lump_prop(educa_jefe, prop = 0.10, other_level = "Otros")
  )
```

Revisamos cuáles quedaron.

```{r}
#| eval: false
concentrado %>% tabyl(educa_agrupada)
```

Una vez agrupadas las categorías con menos del 10%, podemos etiquetar las más frecuentes con `fct_recode()`:

```{r}
#| eval: false
concentrado <- concentrado %>%
  mutate(
    educa_agrupada = fct_recode(educa_agrupada,
      "Primaria incompleta"  = "03",
      "Primaria completa"    = "04",
      "Secundaria completa"  = "06",
      "Preparatoria completa" = "08",
      "Profesional completa" = "10"
    )
  )
```

La vemos.
```{r}
#| eval: false
concentrado %>% tabyl(educa_agrupada)
```

## Fechas

### Extraer componentes de una fecha

La variable `fecha_inicio` viene como `character`. ¿Qué pasa si sólo tuviéramos la variable fecha_inicio y queremos extraer el año, mes y día para generar tres variables distintas? Antes de extraer cualquier componente, debemos convertirla al tipo `Date` con `ymd()`, que asume el orden Año-Mes-Día:

```{r}
#| eval: false
victimas <- victimas %>% 
  mutate(fecha_inicio = ymd(fecha_inicio))
```

La función ymd() de lubridate convierte en texto la fecha, pero asumiendo que el formato está en orden Year-Month-Day (como viene originalmente).

Ahora sí, queremos extraer el año, mes y día a partir de fecha_inicio.

```{r}
#| eval: false
victimas <- victimas %>% 
  mutate(
    annio = year(fecha_inicio),   # Año
    mes   = month(fecha_inicio),  # Mes (número)
    dia   = day(fecha_inicio)     # Día
  )
```


Los objetos `Date` en R se almacenan como números enteros (días transcurridos desde cierta fecha). El formato que vemos al imprimir es solo una representación. Esto significa que **el orden día/mes/año no existe dentro del objeto**; solo existe al momento de imprimirlo o exportarlo.

`format()` convierte una fecha a `character` con el orden deseado:

```{r}
#| eval: false
victimas <- victimas %>%
  mutate(
    fecha_formateada1 = format(fecha_inicio, "%d-%m-%Y"),  # día-mes-año
    fecha_formateada2 = format(fecha_inicio, "%m/%d/%Y"),  # mes/día/año
    fecha_formateada3 = format(fecha_inicio, "%Y/%m/%d")   # año/mes/día
  )
```

### Cálculo de duraciones

No necesitamos cambiar formatos para hacer cálculos. `difftime()` calcula la diferencia entre dos fechas directamente sobre objetos `Date`:

```{r}
#| eval: false
# Convertir fecha_hecho a Date
victimas <- victimas %>%
  mutate(fecha_hecho = ymd(fecha_hecho))

# Calcular días entre el hecho y el inicio de la carpeta
victimas <- victimas %>%
  mutate(
    diferencia_dias = as.numeric(difftime(fecha_inicio, fecha_hecho, units = "days"))
  )
```

### Creación de variables de fecha

Si tenemos el mes y el año en columnas distintas, podemos combinarlas con `make_date()`. Como esta función requiere al menos día, mes y año, asigna el día 1 por defecto cuando solo se proveen mes y año:

```{r}
#| eval: false
victimas <- victimas %>%
  mutate(fecha_mes_año = make_date(month = mes, year = annio))
# Reformatear para mostrar solo mes y año
victimas <- victimas %>%
  mutate(fecha_mes_año = format(fecha_mes_año, "%m/%Y"))
```

Si tenemos la fecha en una columna y la hora en otra, las unimos con `paste()` antes de convertir:

```{r}
#| eval: false
victimas <- victimas %>%
  mutate(fecha_hora = ymd_hms(paste(fecha_inicio, hora_inicio)))

class(victimas$fecha_hora)

```

## (Optativo) Etiquetas provenientes de Stata con `{haven}`

Cuando los datos provienen de archivos `.dta` (Stata), las variables suelen traer etiquetas de valor (*value labels*). `{haven}` las preserva como atributos del objeto en R.

### Leer el archivo y revisar etiquetas

Si tuviéramos que importar bases en formato Stata:
```{r}
#| eval: false
lapop_dta <- read_dta(here("data", "raw", "mex_lapop_2023.dta"))
```

Podemos ver las etiquetas de la variable

```{r}
#| eval: false
attr(lapop_dta$ur, "labels")
```

### Convertir etiquetas a factor

as_factor() convierte las etiquetas Stata en niveles de factor en R
```{r}
#| eval: false
lapop_dta <- lapop_dta %>%
  mutate(ur = as_factor(ur))
```

Veamos los niveles.

```{r}
#| eval: false
levels(lapop_dta$ur)
```

::: callout-note
## `as_factor()` de `{haven}`

A diferencia de `as.factor()` de R base, `haven::as_factor()` usa las **etiquetas** del archivo Stata como niveles del factor (en lugar de los valores numéricos). Esto facilita enormemente el trabajo con microdatos del INEGI distribuidos en formato Stata.
:::
