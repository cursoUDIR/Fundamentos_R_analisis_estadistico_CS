recodificar_lugar <- function(x) {

morelia <- morelia |>

5 Tablas de frecuencia: R base vs. {janitor} vs. {dplyr}

5.1 Percepción general de seguridad en Morelia

5.1.1 R base: table() y prop.table()

# Frecuencias absolutas
table(morelia$seg_ciudad)

# Frecuencias relativas (proporciones)
prop.table(table(morelia$seg_ciudad))

# Porcentajes redondeados
round(prop.table(table(morelia$seg_ciudad)) * 100, 1)

5.1.2 {janitor}: tabyl() + adorn_*()

tabyl() produce frecuencias absolutas y relativas. Las funciones adorn_*() agregan totales, porcentajes con formato y etiquetas.

morelia |>
  tabyl(seg_ciudad) |>
  adorn_totals("row") |>
  adorn_pct_formatting(digits = 1) |>
  knitr::kable(
    caption   = "Percepción de seguridad en Morelia · ENSU T4 2025",
    col.names = c("Percepción", "n", "%")
  )
Nota¿Qué hace cada adorn_*()?
  • adorn_totals("row"): agrega una fila de totales.
  • adorn_percentages(): convierte frecuencias a proporciones (por defecto sobre el total de filas).
  • adorn_pct_formatting(digits = 1): da formato de porcentaje con un decimal.
  • adorn_ns(): muestra también los valores absolutos junto a los porcentajes.

5.1.3 {dplyr}: count() + mutate()

morelia |>
  count(seg_ciudad, name = "n") |>
  mutate(
    pct = round(n / sum(n) * 100, 1)
  ) |>
  knitr::kable(
    caption  = "Percepción de seguridad en Morelia · ENSU T4 2025",
    col.names = c("Percepción", "n", "%")
  )

6.1 Comparación en una sola tabla con pivot_longer()

Podemos comparar varios lugares y reestructurar a formato long.

morelia |>
  select(seg_calle, seg_transporte) |>
  pivot_longer(
    cols      = everything(),
    names_to  = "lugar",
    values_to = "percepcion"
  ) |>
  mutate(
    lugar = recode(lugar,
      "seg_calle"      = "Calle",
      "seg_transporte" = "Transporte público"
    )
  ) |>
  filter(!is.na(percepcion)) |>
  count(lugar, percepcion) |>
  group_by(lugar) |>
  mutate(pct = round(n / sum(n) * 100, 1)) |>
  ungroup() |>
  pivot_wider(
    names_from  = percepcion,
    values_from = c(n, pct)
  ) |>
  knitr::kable(caption = "Percepción de seguridad por lugar · Morelia · ENSU T4 2025")

7 Expectativa sobre la delincuencia

7.1 Tabla con frecuencias acumuladas

Para variables ordinales (como escalas de Likert), las frecuencias acumuladas ayudan a interpretar la distribución.

morelia |>
  tabyl(expectativa) |>
  adorn_totals("row") |>
  adorn_pct_formatting(digits = 1) |>
  knitr::kable(caption = "Expectativa sobre delincuencia en los próximos 12 meses · Morelia")
morelia |>
  count(expectativa) |>
  filter(!is.na(expectativa)) |>
  mutate(
    pct     = round(n / sum(n) * 100, 1),
    pct_acum = cumsum(pct)
  ) |>
  knitr::kable(
    caption   = "Expectativa sobre delincuencia — frecuencias acumuladas · Morelia",
    col.names = c("Expectativa", "n", "%", "% acumulado")
  )
Nota¿Cuándo usar frecuencias acumuladas?

Las frecuencias acumuladas son útiles en variables ordinales (donde el orden importa), como escalas de satisfacción, efectividad o confianza. Permiten responder preguntas como: ¿qué porcentaje considera que la situación mejorará o seguirá igual de bien?

8 Desempeño y confianza en autoridades

8.1 Comparación de efectividad entre autoridades

morelia |>
  select(efect_mun, efect_estatal, efect_gn) |>
  pivot_longer(
    cols      = everything(),
    names_to  = "autoridad",
    values_to = "efectividad"
  ) |>
  mutate(
    autoridad = recode(autoridad,
      "efect_mun"     = "Policía Municipal",
      "efect_estatal" = "Policía Estatal",
      "efect_gn"      = "Guardia Nacional"
    )
  ) |>
  filter(!is.na(efectividad)) |>
  count(autoridad, efectividad) |>
  group_by(autoridad) |>
  mutate(pct = round(n / sum(n) * 100, 1)) |>
  ungroup() |>
  select(-n) |>
  pivot_wider(names_from = efectividad, values_from = pct) |>
  gt() |>
  tab_header(
    title    = "Efectividad percibida de las autoridades de seguridad (%)",
    subtitle = "Morelia · ENSU T4 2025"
  ) |>
  tab_spanner(
    label   = "Evaluación (%)",
    columns = c(`Poco/nada efectivo`, `Muy/algo efectivo`)
  ) |>
  cols_label(autoridad = "Autoridad") |>
  tab_source_note("Fuente: ENSU T4 2025, INEGI. Porcentajes sobre respuestas válidas.")

9 Ordenamiento de categorías con {forcats}

A veces es útil ordenar las categorías por frecuencia para facilitar la lectura.

9.1 fct_reorder(): ordenar por frecuencia

morelia |>
  count(efect_mun) |>
  filter(!is.na(efect_mun)) |>
  mutate(
    efect_mun = fct_reorder(efect_mun, n, .desc = TRUE)
  ) |>
  arrange(efect_mun) |>
  knitr::kable(caption = "Efectividad Policía Municipal (ordenada por frecuencia)")

9.2 fct_lump(): colapsar categorías poco frecuentes

Cuando hay categorías con muy pocos casos, fct_lump() las agrupa en “Otro”.

Ejemplo con expectativa: conservar solo las 2 categorías más frecuentes.

morelia |>
  mutate(
    expectativa_agrup = fct_lump(expectativa, n = 2, other_level = "Otras respuestas")
  ) |>
  tabyl(expectativa_agrup) |>
  adorn_totals("row") |>
  adorn_pct_formatting(digits = 1) |>
  knitr::kable(caption = "Expectativa sobre delincuencia (categorías agrupadas)")
Tipfct_lump() vs. fct_lump_prop()
  • fct_lump(n = 2): conserva las n categorías más frecuentes.
  • fct_lump_prop(prop = 0.05): conserva las categorías que superan el 5% del total. Ambas son útiles para limpiar tablas con muchas categorías residuales.

10 Resumen comparativo: R base vs. {janitor} vs. {dplyr}

Operación R base {janitor} {dplyr}
Frecuencias absolutas table(df$var) tabyl(var) count(var)
Frecuencias relativas prop.table(table(df$var)) tabyl() (incluye valid_percent) count() |> mutate(pct = n/sum(n))
Agregar total addmargins(table(df$var)) adorn_totals("row") manual con bind_rows()
Formato de porcentaje round(...*100, 1) adorn_pct_formatting(digits=1) scales::percent()
Frecuencias acumuladas cumsum(prop.table(...)) manual con mutate(cumsum(...)) mutate(pct_acum = cumsum(pct))
Tabla lista para reporte requiere formato extra kable() directo gt() con opciones

## Tablas de frecuencia: R base vs. {janitor} vs. {dplyr}

### Percepción general de seguridad en Morelia

#### R base: `table()` y `prop.table()`

```{r}
#| eval: false
# Frecuencias absolutas
table(morelia$seg_ciudad)

# Frecuencias relativas (proporciones)
prop.table(table(morelia$seg_ciudad))

# Porcentajes redondeados
round(prop.table(table(morelia$seg_ciudad)) * 100, 1)
```

#### {janitor}: `tabyl()` + `adorn_*()`

`tabyl()` produce frecuencias absolutas y relativas. Las funciones `adorn_*()` agregan totales, porcentajes con formato y etiquetas.

```{r}
#| eval: false
morelia |>
  tabyl(seg_ciudad) |>
  adorn_totals("row") |>
  adorn_pct_formatting(digits = 1) |>
  knitr::kable(
    caption   = "Percepción de seguridad en Morelia · ENSU T4 2025",
    col.names = c("Percepción", "n", "%")
  )
```

::: {.callout-note}
## ¿Qué hace cada `adorn_*()`?

- `adorn_totals("row")`: agrega una fila de totales.
- `adorn_percentages()`: convierte frecuencias a proporciones (por defecto sobre el total de filas).
- `adorn_pct_formatting(digits = 1)`: da formato de porcentaje con un decimal.
- `adorn_ns()`: muestra también los valores absolutos junto a los porcentajes.
:::

#### {dplyr}: `count()` + `mutate()`

```{r}
#| eval: false
morelia |>
  count(seg_ciudad, name = "n") |>
  mutate(
    pct = round(n / sum(n) * 100, 1)
  ) |>
  knitr::kable(
    caption  = "Percepción de seguridad en Morelia · ENSU T4 2025",
    col.names = c("Percepción", "n", "%")
  )
```

morelia |>

## Ordenamiento de categorías con `{forcats}`

A veces es útil ordenar las categorías por frecuencia para facilitar la lectura.

### `fct_reorder()`: ordenar por frecuencia

```{r}
#| eval: false
morelia |>
  count(efect_mun) |>
  filter(!is.na(efect_mun)) |>
  mutate(
    efect_mun = fct_reorder(efect_mun, n, .desc = TRUE)
  ) |>
  arrange(efect_mun) |>
  knitr::kable(caption = "Efectividad Policía Municipal (ordenada por frecuencia)")
```

### `fct_lump()`: colapsar categorías poco frecuentes

Cuando hay categorías con muy pocos casos, `fct_lump()` las agrupa en "Otro".

Ejemplo con expectativa: conservar solo las 2 categorías más frecuentes.
```{r}
#| eval: false
morelia |>
  mutate(
    expectativa_agrup = fct_lump(expectativa, n = 2, other_level = "Otras respuestas")
  ) |>
  tabyl(expectativa_agrup) |>
  adorn_totals("row") |>
  adorn_pct_formatting(digits = 1) |>
  knitr::kable(caption = "Expectativa sobre delincuencia (categorías agrupadas)")
```

::: {.callout-tip}
## `fct_lump()` vs. `fct_lump_prop()`
- `fct_lump(n = 2)`: conserva las *n* categorías más frecuentes.
- `fct_lump_prop(prop = 0.05)`: conserva las categorías que superan el 5% del total.
Ambas son útiles para limpiar tablas con muchas categorías residuales.
:::

## Resumen comparativo: R base vs. {janitor} vs. {dplyr}

| Operación                        | R base                              | {janitor}                          | {dplyr}                              |
|----------------------------------|-------------------------------------|------------------------------------|--------------------------------------|
| Frecuencias absolutas            | `table(df$var)`                     | `tabyl(var)`                       | `count(var)`                         |
| Frecuencias relativas            | `prop.table(table(df$var))`         | `tabyl()` (incluye `valid_percent`)| `count() |> mutate(pct = n/sum(n))`  |
| Agregar total                    | `addmargins(table(df$var))`         | `adorn_totals("row")`              | manual con `bind_rows()`             |
| Formato de porcentaje            | `round(...*100, 1)`                 | `adorn_pct_formatting(digits=1)`   | `scales::percent()`                  |
| Frecuencias acumuladas           | `cumsum(prop.table(...))`           | manual con `mutate(cumsum(...))`   | `mutate(pct_acum = cumsum(pct))`     |
| Tabla lista para reporte         | requiere formato extra              | `kable()` directo                  | `gt()` con opciones                  |
